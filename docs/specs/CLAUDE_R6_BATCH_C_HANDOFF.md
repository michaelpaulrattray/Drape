## 3–9. Architecture and behavior (summary)

Covered in D-56 and §2 above; the load-bearing invariants, each with a named test:

- **Ordering (R2):** classification → normalization → refusal ALL complete before `createGeneration`, deduction, and image calls at every door; every refusal class is zero-net-charge with no rows/writes/calls (`typedIterationDoors` "shared-authority refusals…", `batchC-doors`, `batchC-structured` M22 ordering tests — the three deduct-before-parse creation paths are reordered).
- **Identity:** draft + authoritative `frontClose` only; normalized typed patch; atomic commit (document/prefs/schema + anchor + new revision + stale-all-pinned + typed-edit provenance); commit failure = rollback + exactly one refund (`typedIterationDoors` atomic-commit + M20 tests).
- **Presentation / marks / eyelashes:** deterministic refusals with the ratified copy at edit doors and creation intake; marks refuse every family/operation/status/modality during R6.
- **Image-only:** asset-only, documents byte-unchanged, `display` role + current revision, no compaction/reconcile/stale; drafts and minted alike (M17 tests).

- **References:** evidence never authorization; named-leaf-only; vague/whole refusals free; per-leaf modality gates per the ledger; normalized values only (relational prose rejected at the parser); §8.4 single-leaf unlock rides `policyDirectives`, never the raw sentence.
